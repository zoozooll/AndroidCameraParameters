# 데이터 세이프티 가이드

이 문서는 Android Camera Parameters 애플리케이션의 데이터 수집 및 개인정보 보호 관행을 설명합니다.

## 개요

Android Camera Parameters는 "개인정보 보호 우선" 접근 방식으로 설계되었습니다. 진단 도구로서 작동하려면 하드웨어 정보에 액세스해야 하지만 개인 데이터를 수집하거나 전송하지는 않습니다.

## 권한

### 카메라 권한 (`android.permission.CAMERA`)
- **요구 사항**: `CameraManager`에 액세스하고 `CameraCharacteristics`를 가져오는 데 필요합니다.
- **사용**: 앱은 하드웨어 메타데이터만 읽습니다. 사용자의 명시적인 조치(예: 향후 버전에 이미지 캡처 테스트가 추가되는 경우) 없이는 비디오를 녹화하거나 사진을 찍지 **않습니다**.

## 데이터 수집

- **개인 정보**: 앱은 이름, 이메일 주소, 전화번호 또는 기타 개인 식별 정보를 **수집하지 않습니다**.
- **위치 데이터**: 앱은 사용자의 GPS 또는 네트워크 위치에 **액세스하지 않습니다**.
- **하드웨어 메타데이터**: 앱은 카메라 렌즈의 기술 사양(해상도, 초점 거리, 지원 모드)을 읽습니다. 이 데이터는 사용자가 명시적으로 "JSON 내보내기" 기능을 사용하여 공유하지 않는 한 기기에 남아 있습니다.

## 데이터 공유

애플리케이션은 데이터를 제3자와 **공유하지 않습니다**. 핵심 앱에는 추적 SDK(Firebase Analytics 또는 Facebook SDK 등)가 통합되어 있지 않습니다.

## 사용자 제어

- **JSON 내보내기**: 사용자는 원시 카메라 파라미터 JSON을 복사하거나 공유하도록 선택할 수 있습니다. 이는 전적으로 사용자가 시작한 것입니다.
- **권한**: Android 시스템 설정을 통해 언제든지 카메라 권한을 취할 수 있지만, 권한이 없으면 앱에서 카메라 세부 정보를 표시할 수 없습니다.
