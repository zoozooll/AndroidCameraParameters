# Руководство по безопасности данных

В этом документе описаны методы сбора данных и обеспечения конфиденциальности в приложении Android Camera Parameters.

## Обзор

Приложение Android Camera Parameters разработано с подходом «Конфиденциальность прежде всего». Как диагностический инструмент, оно нуждается в доступе к информации об оборудовании для работы, но оно не собирает и не передает личные данные.

## Разрешения

### Разрешение на использование камеры (`android.permission.CAMERA`)
- **Требование**: необходимо для доступа к `CameraManager` и получения `CameraCharacteristics`.
- **Использование**: приложение читает только метаданные оборудования. Оно **не записывает** видео и не делает фотографии без явного действия пользователя (например, в будущих версиях, если будет добавлено тестирование захвата изображений).

## Сбор данных

- **Личная информация**: приложение **не собирает** имена, адреса электронной почты, номера телефонов или любые другие личные идентификаторы.
- **Данные о местоположении**: приложение **не имеет доступа** к вашему GPS или местоположению по сети.
- **Метаданные оборудования**: приложение считывает технические характеристики линз вашей камеры (разрешение, фокусное расстояние, поддерживаемые режимы). Эти данные остаются на вашем устройстве, если вы явно не воспользуетесь функцией «Экспорт в JSON», чтобы поделиться ими.

## Обмен данными

Приложение **не передает** никакие данные третьим лицам. В основное приложение не интегрированы никакие SDK для отслеживания (например, Firebase Analytics или Facebook SDK).

## Управление пользователем

- **Экспорт в JSON**: пользователи могут скопировать или поделиться необработанным JSON параметров камеры. Это действие полностью инициируется пользователем.
- **Разрешения**: вы можете отозвать разрешение на использование камеры в любое время через системные настройки Android, хотя приложение не сможет отображать сведения о камере без него.
