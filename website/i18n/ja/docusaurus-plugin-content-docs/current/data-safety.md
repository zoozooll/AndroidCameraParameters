# データセーフティガイド

このドキュメントでは、Android カメラパラメータアプリケーションのデータ収集とプライバシー慣行の概要を説明します。

## 概要

Android カメラパラメータは、「プライバシーファースト」のアプローチで設計されています。診断ツールとして、機能するためにハードウェア情報にアクセスする必要がありますが、個人データを収集または送信することはありません。

## 権限

### カメラ権限 (`android.permission.CAMERA`)
- **要件**: `CameraManager` にアクセスし、`CameraCharacteristics` を取得するために必要です。
- **使用法**: アプリはハードウェアメタデータのみを読み取ります。ユーザーの明示的なアクション (例: 将来のバージョンで画像キャプチャテストが追加された場合) がない限り、ビデオの録画や写真の撮影は**行いません**。

## データ収集

- **個人情報**: アプリは、氏名、メールアドレス、電話番号、またはその他の個人識別情報を**収集しません**。
- **位置データ**: アプリは、GPS またはネットワークの位置情報に**アクセスしません**。
- **ハードウェアメタデータ**: アプリはカメラレンズの技術仕様 (解像度、焦点距離、サポートされているモード) を読み取ります。このデータは、ユーザーが「JSON をエクスポート」機能を使用して明示的に共有しない限り、デバイスに留まります。

## データ共有

アプリケーションは、データを第三者と**共有しません**。コアアプリには、トラッキング SDK (Firebase Analytics や Facebook SDK など) は統合されていません。

## ユーザーコントロール

- **JSON エクスポート**: ユーザーは生のカメラパラメータ JSON をコピーまたは共有することを選択できます。これは完全にユーザー主導です。
- **権限**: Android のシステム設定からいつでもカメラの権限を取り消すことができますが、権限がないとアプリはカメラの詳細を表示できなくなります。
