# 資料安全指南

本文件概述了 Android 相機參數應用程式的資料收集與隱私實踐。

## 概覽

Android 相機參數採用「隱私至上」的方法設計。作為一款診斷工具，它需要存取硬體資訊才能運作，但它不會收集或傳輸個人資料。

## 權限

### 相機權限 (`android.permission.CAMERA`)
- **要求**：存取 `CameraManager` 並擷取 `CameraCharacteristics` 所必需。
- **用法**：應用程式僅讀取硬體中繼資料。除非使用者採取明確行動（例如，在未來版本中加入影像擷取測試），否則它**不會**錄製影片或拍攝照片。

## 資料收集

- **個人資訊**：應用程式**不收集**姓名、電子郵件地址、電話號碼或任何其他個人識別資訊。
- **地點資料**：應用程式**不存取**您的 GPS 或網路地點。
- **硬體中繼資料**：應用程式讀取相機鏡頭的技術規格（解析度、焦距、支援模式）。除非您明確使用「匯出 JSON」功能來分享，否則此資料將保留在您的設備上。

## 資料分享

本應用程式**不與第三方分享**任何資料。核心應用程式中未整合任何追蹤 SDK（如 Firebase Analytics 或 Facebook SDK）傳合。

## 使用者控制

- **JSON 匯出**：使用者可以選擇複製或分享原始相機參數 JSON。這完全由使用者發起。
- **權限**：您可以隨時透過 Android 系統設定撤銷相機權限，但若沒有該權限，應用程式將無法顯示相機詳細資訊。
